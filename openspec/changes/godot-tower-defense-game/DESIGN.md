# Godot 4.6 2D 塔防游戏 - 设计文档

## 项目概述

一个基于 Godot 4.6 的 2D 塔防游戏，核心特色是开放地图玩家自建路径系统。

## 核心玩法

### 游戏流程
1. 玩家获得初始金币
2. 开始波次，敌人从起点生成
3. 玩家使用金币建造/升级防御塔
4. 击杀敌人获得金币奖励
5. 敌人到达终点扣除生命值
6. 通过所有波次进入下一关

### 胜负条件
- **胜利**：完成当前关卡所有波次
- **失败**：生命值归零

## 系统设计

### 1. 地图系统
- 网格布局：20x15 格子
- 固定起点和终点
- 可建造区域和不可建造区域
- 使用 TileMap 实现

### 2. 寻路系统
- A* 算法实现
- 飞行单位：遵循寻路但无视地面障碍（可飞过防御塔）
- 地面单位：必须绕过防御塔
- 建塔时验证路径可达性（防止堵死）

### 3. 防御塔系统

#### 塔类型
1. **基础炮塔**
   - 单体伤害
   - 中等射程和攻速
   - 成本：100 金币

2. **减速塔**
   - 降低敌人 50% 移动速度
   - 短射程
   - 成本：150 金币

3. **范围塔**
   - AOE 伤害
   - 攻击范围内所有敌人
   - 成本：200 金币

4. **狙击塔**
   - 高伤害，长射程
   - 攻速慢
   - 成本：250 金币

5. **辅助塔**（可选）
   - 增强周围塔 25% 攻击力
   - 不直接攻击
   - 成本：180 金币

### 4. 敌人系统

#### 敌人类型
1. **快速单位**
   - 移动速度：150
   - 血量：50
   - 奖励：10 金币

2. **坦克单位**
   - 移动速度：50
   - 血量：300
   - 奖励：30 金币

3. **飞行单位**
   - 移动速度：100
   - 血量：80
   - 可飞过防御塔
   - 奖励：20 金币

4. **再生单位**
   - 移动速度：80
   - 血量：150
   - 每秒回复 5 HP
   - 奖励：25 金币

### 5. 资源系统
- 初始金币：500
- 初始生命值：20
- 击杀敌人获得金币
- 每波开始额外奖励：50 金币

### 6. 波次系统
- 每关 10 波
- 波次间隔：5 秒
- 敌人数量和类型随波次递增

## 技术架构

### 场景结构
```
Main.tscn (主场景)
├── UI (CanvasLayer)
│   ├── GoldLabel
│   ├── HealthLabel
│   ├── WaveLabel
│   └── TowerButtons
├── GameMap (TileMap)
├── PathManager (Node)
├── TowerManager (Node)
├── EnemySpawner (Node)
└── Projectiles (Node2D)
```

### 核心脚本
- `game_manager.gd` - 游戏状态管理
- `pathfinding.gd` - A* 寻路算法
- `tower_base.gd` - 防御塔基类
- `enemy_base.gd` - 敌人基类
- `projectile.gd` - 子弹/攻击效果

### 占位符美术
- 防御塔：ColorRect (不同颜色)
- 敌人：ColorRect (不同颜色和大小)
- 地图：TileMap (简单色块)
- 子弹：小圆形 Sprite2D

## 开发优先级

### Phase 1: 核心系统
1. 地图网格系统
2. A* 寻路实现
3. 基础敌人生成和移动

### Phase 2: 战斗系统
4. 基础炮塔实现
5. 攻击和伤害系统
6. 资源和生命值管理

### Phase 3: 扩展内容
7. 其他塔类型
8. 特殊敌人能力
9. 波次系统

### Phase 4: 完善
10. UI 优化
11. 关卡设计
12. 平衡性调整
